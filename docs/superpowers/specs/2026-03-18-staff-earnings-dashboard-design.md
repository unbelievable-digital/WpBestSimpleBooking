# Staff Earnings Dashboard & Self-Booking — Design Spec

**Date:** 2026-03-18
**Status:** Approved
**Approach:** Approach 1 — Extend existing tables with new payments table

---

## Overview

Extend the existing staff portal with earnings tracking, performance metrics, and admin-side payment management. Additionally, allow staff members to create bookings for themselves with a streamlined interface.

---

## 1. Database

### New Table: `unbsb_staff_payments`

```sql
CREATE TABLE {prefix}unbsb_staff_payments (
    id BIGINT(20) UNSIGNED NOT NULL AUTO_INCREMENT,
    staff_id BIGINT(20) UNSIGNED NOT NULL,
    amount DECIMAL(10,2) NOT NULL,
    payment_date DATE NOT NULL,
    payment_method VARCHAR(50) DEFAULT NULL,
    notes TEXT,
    recorded_by BIGINT(20) UNSIGNED NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    KEY staff_id (staff_id),
    KEY payment_date (payment_date)
);
```

### Balance Calculation

- **Total Earnings** = `SUM(amount)` from `unbsb_staff_earnings` WHERE `staff_id` = X
- **Total Paid** = `SUM(amount)` from `unbsb_staff_payments` WHERE `staff_id` = X
- **Remaining Balance** = Total Earnings - Total Paid

Period filtering (this month, last month, date range) applies to both tables.

**Filter logic per table:**
- `unbsb_staff_earnings`: Filter by `period` column (format `Y-m`), e.g., `WHERE period = '2026-03'`
- `unbsb_staff_payments`: Filter by `payment_date` column (format `DATE`), e.g., `WHERE payment_date BETWEEN '2026-03-01' AND '2026-03-31'`

**Note:** Summary cards "Total Earnings (all time)" and "Total Paid" ignore period filter — they always show all-time totals. Only the detail tables below them respect the period filter.

## 5. Commission Calculation Logic

Uses existing staff fields: `salary_type`, `salary_percentage`, `salary_fixed`.

**Actual code behavior** (from `calculate_commission()` in `class-unbsb-staff.php`):

| salary_type | Calculation | Example |
|-------------|-------------|---------|
| `percentage` | Booking total × percentage | 500₺ × 30% = 150₺ |
| `fixed` | Returns 0 — fixed salary is a monthly lump sum, not per-booking | Monthly salary recorded via `record_salary()` |
| `mix` | Booking total × percentage (only percentage part) | 500₺ × 20% = 100₺ |

**Important:** For `fixed` type staff, earnings come from monthly `salary` type records, not `commission` type. The dashboard earnings query must `SUM(amount)` across both `type = 'commission'` AND `type = 'salary'` rows in `unbsb_staff_earnings`.

### When recorded
- **Commission** (`type = 'commission'`): Written to `unbsb_staff_earnings` when booking status changes to `completed`
- **Salary** (`type = 'salary'`): Written via `record_salary()` as monthly lump sum for `fixed` type staff
- Existing behavior, no changes needed to calculation logic

### Cancellation / Edit
- `completed` → `cancelled`: earning record deleted
- Booking price edited: earning recalculated

### Multi-service bookings
- Separate earning record per service (for `percentage` and `mix` types)
- Staff earns commission only on services they provided
- Commission base is booking-level `price` from `unbsb_bookings` (not per-service price from `booking_services`)

## 7. Staff Self-Booking

When a staff member accesses the "New Booking" page:

- **Staff selector disabled** — automatically set to themselves
- **Calendar** shows only their own availability
- **Service list** shows only services they provide
- Everything else remains the same (customer search, date/time selection, save)

### Access Control
- Staff must have `unbsb_view_own_bookings` capability
- Booking is created with their staff_id, cannot be changed
- **Backend enforcement required:** The AJAX/REST handler must verify that the `staff_id` in the request payload matches the current user's linked staff record. Frontend disabling alone is not sufficient.
- Admin notifications still fire normally

### Implementation Approach
- Modify existing `render_new_booking()` — pass a context variable (`$is_staff_self_booking`) to the template
- Template conditionally disables staff selector and pre-filters services
- No separate template file needed

## 10. Performance

### Staff List "Remaining Balance" Column
- Use a single aggregated query with LEFT JOINs to `staff_earnings` and `staff_payments` — not N+1 individual queries
- Query: `SELECT s.*, COALESCE(e.total, 0) - COALESCE(p.total, 0) AS remaining_balance FROM staff s LEFT JOIN (SELECT staff_id, SUM(amount) as total FROM staff_earnings GROUP BY staff_id) e ON ... LEFT JOIN (SELECT staff_id, SUM(amount) as total FROM staff_payments GROUP BY staff_id) p ON ...`
